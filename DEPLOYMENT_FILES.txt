🚀 QUICK COMMANDS
================================================================================

Configuration:
  aws configure                                    # Setup AWS credentials
  echo $AWS_REGION                                 # Check region

Deployment:
  ./quick-deploy.sh                                # Automatic deployment
  ./deploy.sh                                      # Interactive menu

Verification:
  kubectl get pods -n yelp                         # View all pods
  kubectl get svc -n yelp                          # View all services
  kubectl cluster-info                             # Cluster info
  kubectl get nodes                                # Node status

Testing:
  kubectl port-forward svc/user-service 8001:8001 -n yelp &
  curl http://localhost:8001/health                # Test endpoint

Monitoring:
  kubectl logs deployment/user-service -n yelp     # View logs
  kubectl logs -f deployment/user-service -n yelp  # Stream logs
  kubectl describe pod <pod-name> -n yelp          # Debug pod

Cleanup:
  kubectl delete namespace yelp                    # Delete namespace
  eksctl delete cluster --name yelp-cluster        # Delete cluster
  aws ecr delete-repository --repository-name yelp/user-service --force

================================================================================

Issue: Pods stuck in ImagePullBackOff
→ Solution: Check ECR login, verify image URLs in manifests

Issue: Services can't communicate
→ Solution: Test DNS with: kubectl exec -it <pod> -- nslookup mongodb

Issue: Frontend can't load
→ Solution: Check LoadBalancer status, verify CORS settings

Issue: Database connection failing
→ Solution: Port forward MongoDB, test with mongosh

📞 COMMON QUESTIONS
================================================================================

Q: How long does deployment take?
A: 60-90 minutes (mostly automated, ~15 min wait for EKS cluster)

Q: How much will it cost?
A: $5-10 for 1-2 hours of testing. Delete cluster after testing.

Q: Can I test locally first?
A: Yes! Run `docker-compose up` before deploying to AWS.

Q: What if deployment fails?
A: Check logs with `kubectl logs`, see troubleshooting guide.

Q: Can I modify the deployment?
A: Yes! Edit manifests in k8s/ directory and reapply.

Q: Is my data safe?
A: Yes! MongoDB data stored in AWS EBS volumes with persistence.

Q: Can I scale services?
A: Yes! `kubectl scale deployment/user-service --replicas=3 -n yelp`

Q: Do I need to redeploy every time?
A: No! Only for code changes. Rerun scripts to update images.
